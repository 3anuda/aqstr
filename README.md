# Aqmola Start


## Технологический стек

Использовались проверенные инструменты для быстрой разработки MVP:

* **Backend:** PHP (Native) — собственный REST API.
* **Database:** MySQL (phpMyAdmin) — хранение данных.
* **Mobile:** Flutter — кроссплатформенное приложение (WebView интеграция).
* **Frontend:** HTML5, Bootstrap 5, Custom CSS.
* **Bot:** Python 3 + Aiogram.

---

## Инструкция по запуску

### Шаг 1. Сервер и База Данных (XAMPP)
Проект работает на локальном сервере Apache.

1.  Установите **XAMPP**.
2.  Запустите **Apache** и **MySQL** в панели управления.
3.  Откройте `http://localhost/phpmyadmin`.
4.  Создайте базу данных с именем `aqmola_start`.
5.  Импортируйте файл `database.sql` (находится в корне репозитория), чтобы создать таблицы и заполнить их тестовыми данными.
6.  Скопируйте папку проекта `AqmolaStart` в папку `C:/xampp/htdocs/`.

### Шаг 2. Настройка IP (Важно!)
Для того чтобы мобильное приложение и бот видели сервер, нужно использовать ваш локальный IP (например, `192.168.x.x` или внешний статический IP), а не `localhost`.

1.  Узнайте ваш IP (команда `ipconfig` в терминале).
2.  Откройте файлы `index.html` (в корне и в папке `app/`) и укажите ваш IP в переменной `API_URL`.

### Шаг 3. Запуск Telegram-бота
Бот написан на Python3.12

1.  Установите зависимости:
    ```bash
    pip install aiogram requests
    ```
2.  В файле `bot/bot.py` укажите токен вашего бота и IP сервера.
3.  Запустите бота:
    ```bash
    python bot/bot.py
    ```

### Шаг 4. Мобильное приложение
В репозитории лежит исходный код Flutter проекта.
* В main.dart вписать ваш IP
* Готовый APK файл для установки моем IP: `release/AqmolaStart.apk`
* Для сборки из исходников:
    ```bash
    flutter pub get
    flutter build apk --release
    ```

## Функционал MVP

* **Каталог стартапов:** Просмотр карточек проектов с фильтрацией.
* **Календарь событий:** Список мероприятий с датами.
* **Команды:** Просмотр профилей команд и их стека технологий.
* **Админ-панель (Web):** Форма добавления новых событий в базу данных.
* **Раздельный UI:**
    * На ПК — классическое верхнее меню.
    * На телефоне — нативная нижняя навигация (Bottom Bar).
